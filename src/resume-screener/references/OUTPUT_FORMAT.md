# Output Format

## Report Template (Chinese)

```
# 简历评估报告

## 候选人概览
- **姓名**: [姓名]
- **当前职位**: [当前或最近职位]
- **相关经验年限**: [年数]
- **目标岗位**: [JD 岗位名称]

---

## 综合评级

### 总评: [等级]（[标签] - [分数]/10）

[3-5 句总体评价]

### 招聘建议: [强烈推荐面试 / 推荐面试 / 待定 / 不推荐]

[1-2 句建议理由]

---

## 各维度评分

| # | 评估维度 | 等级 | 权重 | 关键证据 |
|---|----------|------|------|----------|
| 1 | 学历背景 | [等级] | [高/中/低] | [简要证据] |
| 2 | 工作经验 | [等级] | [高/中/低] | [简要证据] |
| 3 | 技术技能 | [等级] | [高/中/低] | [简要证据] |
| 4 | 项目经验 | [等级] | [高/中/低] | [简要证据] |
| 5 | 领导能力 | [等级] | [高/中/低] | [简要证据] |
| 6 | 行业知识 | [等级] | [高/中/低] | [简要证据] |
| 7 | 职业发展 | [等级] | [高/中/低] | [简要证据] |
| 8 | 证书成就 | [等级] | [高/中/低] | [简要证据] |
| 9 | 沟通软技能 | [等级] | [高/中/低] | [简要证据] |

---

## 详细分析

### 1. 学历背景匹配度 — [等级]
**岗位要求**: [JD 中的学历要求]
**简历证据**: [具体学历信息]
**评估理由**: [1-3 句分析]

[重复 9 个维度]

---

## 核心优势（Top 3）
1. [优势 + 具体证据]
2. [优势 + 具体证据]
3. [优势 + 具体证据]

## 关注事项（Top 3）
1. [关注点 + 证据或信息缺失说明]
2. [关注点 + 证据或信息缺失说明]
3. [关注点 + 证据或信息缺失说明]

## 面试建议重点
- [方向 1]: [为什么需要深入了解]
- [方向 2]: [为什么需要深入了解]
- [方向 3]: [为什么需要深入了解]

---

## 备注
- [信息不足的说明]
- [做出的假设]
```

## Batch Comparison Format

When evaluating multiple candidates against the same JD, use this summary table:

```
# Candidate Comparison — [Position Name]

| Candidate | Overall | Education | Experience | Skills | Projects | Leadership | Domain | Trajectory | Certs | Communication | Recommendation |
|-----------|---------|-----------|------------|--------|----------|------------|--------|------------|-------|---------------|----------------|
| [Name 1] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Rec] |
| [Name 2] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Grade] | [Rec] |

## Ranking
1. **[Name]** — [Overall Grade] — [One-line summary]
2. **[Name]** — [Overall Grade] — [One-line summary]
```

Then provide individual detailed reports for each candidate.

## Output Language Rule

Match the user's language:
- User communicates in Chinese → Use Chinese template
- User communicates in English → Use English template
- JD is in a different language than user → Still use user's language for the report, translate JD content as needed
